1. OVERVIEW & PURPOSE
================================================================================

This is an interactive educational web application that demonstrates the 
fundamental difference between classical encryption (RSA) and post-quantum 
encryption (PQC/Lattice-based cryptography).

Primary Goal: Show WHY quantum computers threaten current encryption and HOW 
PQC resists quantum attacks through visual, interactive examples.

Key Insight: RSA is deterministic (same input → same output), while PQC is 
probabilistic (same input → different output due to random noise). This 
randomness destroys the mathematical patterns quantum computers exploit.

================================================================================
2. CORE CONCEPTS EXPLAINED
================================================================================

2.1 WHAT IS QUANTUM COMPUTING THREAT?

Quantum computers use principles like superposition and entanglement to solve 
certain problems exponentially faster than classical computers.

Shor's Algorithm (1994):
- Can factor large numbers in polynomial time
- Breaks RSA, ECC, and most current public-key cryptography
- A sufficiently powerful quantum computer could decrypt virtually all 
  current encrypted communications

Grover's Algorithm:
- Provides quadratic speedup for searching unstructured data
- Weakens symmetric encryption (AES-128 becomes effectively 64-bit)

2.2 WHAT IS POST-QUANTUM CRYPTOGRAPHY (PQC)?

PQC refers to cryptographic algorithms designed to be secure against both 
classical and quantum computers. These are based on mathematical problems 
believed hard even for quantum computers.

NIST (National Institute of Standards and Technology) selected four PQC 
algorithms in July 2022:
- CRYSTALS-Kyber: Key encapsulation (general encryption)
- CRYSTALS-Dilithium: Digital signatures
- FALCON: Digital signatures
- SPHINCS+: Hash-based signatures

2.3 WHY RANDOMNESS MATTERS

RSA Structure:
- Based on integer factorization: n = p × q
- Encryption: c = m^e mod n (deterministic)
- Quantum computers can factor n, revealing private key

PQC Approach (LWE):
- Based on Learning With Errors: b = A·s + e (mod q)
- Adds random error 'e' to hide structure
- No known quantum algorithm can efficiently solve this
- Each encryption uses fresh randomness → different ciphertexts

3. MATHEMATICAL FOUNDATIONS
================================================================================

3.1 RSA IMPLEMENTATION IN DEMO

Parameters:
  p = 61, q = 53
  n = p × q = 3233
  φ(n) = (p-1)(q-1) = 60 × 52 = 3120
  e = 17 (public exponent, coprime with φ(n))
  d = 2753 (private exponent: e⁻¹ mod φ(n))

Encryption Formula:
  c = m^e mod n
  Example: 42^17 mod 3233 = 2557

Decryption Formula:
  m = c^d mod n
  Example: 2557^2753 mod 3233 = 42

Key Property: DETERMINISTIC
  Same message always produces same ciphertext

3.2 PQC SIMULATION IN DEMO

Parameters:
  MOD = 65537 (prime number: 2^16 + 1)
  MULTIPLIER = 13 (fixed constant)

Encryption Formula:
  cipher = (message × 13 + random_noise) mod 65537
  where random_noise = random integer between 100 and 2100

Decryption Formula:
  temp = (cipher - stored_noise) mod 65537
  recovered = (temp × modular_inverse(13)) mod 65537

Key Property: PROBABILISTIC
  Same message produces different ciphertexts each time

3.3 THE LWE CONNECTION (Real PQC)

Real Learning With Errors (LWE) Problem:
  Given: many pairs (a_i, b_i) where b_i = a_i·s + e_i (mod q)
  Find: secret s (where e_i are small random errors)

Why It's Hard:
  - No efficient classical algorithm
  - No known quantum speedup
  - Reduces to worst-case lattice problems (SVP, SIVP)
  - Exponential time complexity for both classical and quantum computers

Our Demo Models:
  - Structured part: message × 13 (like a_i·s)
  - Error term: random_noise (like e_i)
  - Finite field: mod 65537 (like mod q)
  - Simplified to 1 dimension (real uses 256+ dimensions)

7. TECHNICAL IMPLEMENTATION
================================================================================

7.1 MODULAR EXPONENTIATION (Binary Method)

Algorithm:
  result = 1
  base = base mod mod
  while exponent > 0:
      if exponent & 1: result = (result × base) mod mod
      base = (base × base) mod mod
      exponent = exponent >> 1
  return result

Time Complexity: O(log exponent)
Space Complexity: O(1)

7.2 EXTENDED EUCLIDEAN ALGORITHM

Used for modular inverse calculation:
  function extendedGCD(a, b):
      if b = 0: return (a, 1, 0)
      (gcd, x1, y1) = extendedGCD(b, a mod b)
      x = y1
      y = x1 - floor(a/b) × y1
      return (gcd, x, y)

7.3 PQC STATE MANAGEMENT

State Object:
  {
    lastNoise: number,        // Random value from last encryption
    lastCiphertext: number,   // Ciphertext from last encryption
    lastPlaintext: number     // Original plaintext
  }

Decryption requires knowing the noise from encryption:
  - Simulates real KEM where decapsulation uses shared secret
  - Ensures encryption and decryption are paired
  - Prevents decryption without recent encryption

7.4 HISTORY TRACKING

9. LIMITATIONS & ACCURACY
================================================================================

9.1 WHAT THIS DEMO DOES NOT DO

Does NOT implement real cryptographic security:
  - RSA uses 1024-bit keys (insecure by modern standards)
  - PQC uses simplified scalar math (not actual lattice operations)
  - No key exchange or key generation
  - No padding schemes (RSA without OAEP is vulnerable)
  - No secure random number generation

Does NOT implement actual PQC algorithms:
  - Not CRYSTALS-Kyber (uses module-LWE, 256+ dimensions)
  - Not FrodoKEM (uses matrix LWE)
  - Not any NIST-approved implementation

Does NOT demonstrate:
  - Performance differences (real PQC is slower)
  - Key sizes (real PQC has larger keys)
  - Network protocol integration
  - Hybrid cryptography (RSA + PQC together)

9.2 HONEST DISCLAIMERS

The demo openly states it is a simulation:
  - "educational simulation" in footer
  - "based on NIST PQC finalists simulation" in table caption
  - Tooltips clarify it's simplified

Why these simplifications are acceptable:
  - Real PQC code is thousands of lines (not educational)
  - Actual lattice math is graduate-level material
  - The core concept (noise hides patterns) is preserved
  - Visual learning requires simplicity

9.3 WHAT IS ACCURATE

Accurate elements:
  - RSA uses correct modular exponentiation
  - PQC concept (adding noise) correctly models LWE principle
  - Deterministic vs probabilistic distinction is accurate
  - NIST algorithm names and status are correct
  - Quantum threat timeline is realistic
  - Shor's algorithm explanation is correct

12. TROUBLESHOOTING
================================================================================

Common Issues and Solutions:

Issue: RSA decryption fails
  Solution: Must encrypt first, ciphertext must be from RSA encryption

Issue: PQC decryption fails
  Solution: Must encrypt immediately before decrypting (state resets)
  Note: Each encryption creates new noise, must decrypt same ciphertext

Issue: Animation not showing
  Solution: Check browser console for errors, ensure CSS loaded

Issue: Numbers too large
  Solution: RSA limited to 0-3232, PQC limited to 0-5000

Issue: Modal not opening
  Solution: Check JavaScript console, ensure element IDs correct

Issue: Responsive layout broken
  Solution: Check viewport meta tag, test at different screen sizes

14. KEY TERMS GLOSSARY
================================================================================

Quantum Computing:
  - Uses qubits (quantum bits) for superposition and entanglement
  - Solves certain problems exponentially faster

Shor's Algorithm:
  - Polynomial-time quantum algorithm for integer factorization
  - Breaks RSA, ECC, and most current public-key crypto

Grover's Algorithm:
  - Quadratic speedup for unstructured search
  - Reduces symmetric key security by half

Lattice-Based Cryptography:
  - Based on hard problems in lattices (LWE, SVP, SIVP)
  - Believed quantum-resistant
  - Basis for CRYSTALS-Kyber, Dilithium, FALCON

Learning With Errors (LWE):
  - Problem: solve b = A·s + e for s
  - Error term e makes it hard
  - Reduces to worst-case lattice problems

Deterministic Encryption:
  - Same input always produces same output
  - RSA is deterministic
  - Creates patterns quantum computers can exploit

Probabilistic Encryption:
  - Same input can produce different outputs
  - Adds randomness to hide patterns
  - PQC is probabilistic

KEM (Key Encapsulation Mechanism):
  - Exchanges symmetric keys securely
  - Kyber is a KEM
  - Alternative to RSA key exchange

NIST:
  - National Institute of Standards and Technology
  - US government agency standardizing cryptography
  - Selected PQC algorithms in 2022
